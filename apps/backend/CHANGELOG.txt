2. FEAT — Async GenAI Calls via Thread Pool Offloading
──────────────────────────────────────────────────────────────
  Files:    app/core/genai_adapter.py
            app/modules/embeddings/service.py
            app/modules/query/service.py
            app/modules/quizzes/service.py
            app/modules/rag_evaluation/service.py
            app/modules/study_buddy/service.py

  Problem: All Gemini API calls (generate_content, embed_content)
           were synchronous, blocking the FastAPI event loop.
           A user request to /query or /study-sessions/{id}/chat
           would stall ALL other API requests until Gemini
           responded (2–20s).

  Changes:
    • genai_adapter.py:
        - Added generate_text_async() — wraps sync SDK calls
          in asyncio.to_thread() so the event loop stays free.
        - Added generate_text_stream_async() — Queue-based SSE
          streaming via thread pool (loop.run_in_executor).
    • embeddings/service.py: Made embed_text() async via thread pool.
    • query/service.py: Made _synthesize() async, all embed_text()
      calls now awaited.
    • study_buddy/service.py: chat() now calls generate_text_async(),
      all embed_text() calls awaited.
    • quizzes/service.py: _call_llm_for_quiz() now async, all
      embed_text() calls awaited.
    • rag_evaluation/service.py: _evaluate_faithfulness() now async,
      _run_faithfulness() correctly awaits it.

  Result: GenAI calls no longer block the event loop. Other API
          requests process freely while Gemini is waiting.


3. FEAT — SSE Streaming Endpoint for Query Synthesis
──────────────────────────────────────────────────────────────
  File:    app/modules/query/router.py

  Changes:
    • Added POST /query/stream — SSE streaming endpoint.
    • Returns event stream: results (retrieved chunks) →
      tokens (streamed synthesis) → done.
    • Event types: results, token, error, done.
    • Uses generate_text_stream_async() for real-time token delivery.
