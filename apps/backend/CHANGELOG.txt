1. BUG FIX — Study Session Retrieval
─────────────────────────────────────
  File:    app/modules/study_buddy/service.py
  Problem: PydanticSerializationError — unable to serialize ObjectId
           when retrieving a session after confirming a journal.
  Fix:     Changed metadata.journal_created to store a plain string
           (journal_id) instead of ObjectId(journal_id).


2. API CHANGE — Journal Endpoints (embedding removed)
──────────────────────────────────────────────────────
  File:    app/modules/journal/dto.py
           app/modules/journal/service.py

  CHANGE:  JournalEntryResponse no longer includes the `embedding`
           vector (list[float]). The 3072-dim vector was only used
           internally by the AI engine and caused response delays.

  Removed field:
    - embedding: list[float]   ← no longer sent to the client

  Kept:
    - embedding_status: str    ← still returned so frontend can poll


3. API CHANGE — PaginatedResponse for ALL list endpoints
─────────────────────────────────────────────────────────
  New file:  app/core/dto.py  ← PaginatedResponse[T] generic DTO

  All three list endpoints now return a wrapped response with
  pagination metadata instead of a bare array:

  ┌─────────────────────────┬─────────────────────────────────────┐
  │ Endpoint                │ Before             │ After          │
  ├─────────────────────────┼─────────────────────────────────────┤
  │ GET /journal-entries/   │ [...]              │ { items,       │
  │                         │                    │   total,       │
  │ GET /topics/            │ [...]              │   skip,        │
  │                         │                    │   limit }      │
  │ GET /sources/           │ [...]              │                │
  └─────────────────────────┴─────────────────────────────────────┘

  The `total` field allows the frontend to build proper page controls.
  ⚠ Frontend must now read response.items instead of the bare array.

5. IMPROVEMENT — Study Buddy Journal Retrieval
───────────────────────────────────────────────
  File:    app/modules/study_buddy/service.py

  Three changes to make the Study Buddy better at reading
  the user's personal journal notes:

  a) Vector search widened: top_k_journals 5 → 12
     (more low-match journal entries are included)

  b) Recent-journals fallback: always fetches the 5 most recent
     journals from the session's topic (or user-wide) and merges
     them with vector results, deduplicated by _id.

  c) Prompt updated: added "HOW TO USE EACH DATA TYPE" section
     that ranks JOURNAL ENTRIES first and instructs the LLM
     to weight personal notes most heavily.
     Response rules now start with:
       "Always start your answer anchored in the user's OWN
        journal entries"


6. PERFORMANCE — Dashboard Optimized
─────────────────────────────────────
  File:    app/modules/dashboard/service.py
           app/modules/dashboard/dto.py

  - Merged 15+ individual count_documents into 3 $facet aggregations
    (journals, chunks, RAG) — reduced DB round-trips by ~60%
  - Added in-memory TTL cache (60s) — most dashboard loads hit cache
  - Removed unused `total_quiz_attempts` field (smaller payload)
  - Removed unused imports (settings, Any)
